# README Image Suggestions

Use assets that are either generated by this repo or clearly licensed for reuse.

## Best images to add

1. Architecture diagram
- Source: generate from Mermaid in `docs/architecture.mmd` or export from the README itself.
- Purpose: show ingestion -> retrieval -> recommendation flow.

2. API screenshot
- Source: FastAPI docs screenshot from `/docs` once the app is running.
- Purpose: show the available endpoints and request/response shapes.

3. Recommendation report preview
- Source: markdown output from `/recommendations/markdown`.
- Purpose: show the stock recommendation digest with sources and rationale.

4. Backtest result chart
- Source: a generated chart from the backtest notebook or future plotting endpoint.
- Purpose: show performance and drawdown visualization.

5. Source coverage panel
- Source: a small custom chart or table showing SEC, web, and Twitter evidence counts.
- Purpose: communicate breadth of ingestion.

## Recommended format

- Prefer SVG for diagrams.
- Prefer PNG for screenshots and charts.
- Keep width under 1400px for GitHub README rendering.

## Safe licensing guidance

- Prefer your own generated screenshots and diagrams.
- If using external logos or photos, verify the license before reuse.
- Avoid scraping random images for the README; use source-owned assets or generated graphics instead.
